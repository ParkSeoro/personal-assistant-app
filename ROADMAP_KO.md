# 만능 비서 시스템 설계안 (한국어)

## 0) 목표
- PC(웹/데스크톱) + Android(모바일 앱) 연동.
- 한국어 음성/텍스트 비서.
- 일정/할 일/자산 대시보드 통합.
- 사용자가 모든 자동화와 권한을 직접 통제.

## 1) 중요한 제약(법/보안/안전)
- 통화기록/문자/SNS 데이터 수집은 **사용자 본인 기기에서, 명시적 권한 허용**이 있는 경우에만 수행.
- 제3자 기기 감시/무단 수집 기능은 제공하지 않음.
- 자동 매매/자동 송금은 기본 비활성화. 2단계 확인 + 일일 손실 한도 + 긴급 정지 스위치 필수.
- 엔진의 "에고"는 페르소나 계층으로만 구현하고, 실제 실행 권한은 정책 엔진이 제한.

## 2) 멀티 엔진 구조
1. 오케스트레이터 엔진(최종 결정권)
2. 일정/생산성 엔진
3. 금융 분석 엔진(조언 중심)
4. 개발 코파일럿 엔진(코드 생성/검토)
5. 음성 인터페이스 엔진(STT/TTS)
6. 리스크/컴플라이언스 엔진(정책/법적 제약)
7. 관찰성 엔진(로그/대시보드/감사 추적)

모든 엔진 출력은 정책 엔진을 거쳐 실행되며, 사용자 승인 규칙이 우선함.

## 3) 실행 플랫폼
- 웹 앱: 현재 저장소 기반 UI.
- Android: Capacitor 래핑 후 APK/AAB 빌드.
- PC 설치 파일:
  - Windows: Electron + electron-builder(NSIS)
  - macOS: dmg
  - Linux: AppImage

## 4) 단계별 개발
### Phase 1 (MVP)
- 한국어 대화, 일정/할 일 위젯, 로컬 암호화 저장, 음성 입력/출력.
- 권한 센터(마이크/위치/알림).
- 활동 로그 화면(어떤 엔진이 무엇을 했는지).

### Phase 2
- 캘린더/메시지 연동 커넥터(사용자 동의 기반).
- 자산 현황 대시보드(수동 입력 + API 연동 선택).
- 전략 백테스트(자동 실행 아님).

### Phase 3
- 자동화 러너(사용자 승인 정책 기반).
- 설치형 로컬 LLM 연결(예: Ollama/vLLM).
- 다중 에이전트 작업 큐와 우선순위 스케줄러.

## 5) 저비용 운영 전략
- 기본 모드: 로컬 LLM + 소형 모델 우선.
- 고비용 추론은 "승인 시에만" 클라우드 호출.
- 캐시/요약/주기적 배치로 토큰 및 API 비용 절감.

## 6) 사용자가 보게 될 통제 기능
- 전역 정지 버튼(Kill Switch)
- 자동화별 ON/OFF
- 데이터 소스별 권한 토글
- 실행 전 승인(고위험 작업)
- 세부 로그/근거/모델 응답 기록

## 7) 다음 구현 우선순위
1. 엔진 레지스트리 + 상태 패널 UI
2. 정책 엔진(승인 규칙)
3. 활동 로그 타임라인
4. Android 패키징 파이프라인
5. 데스크톱 설치파일 빌드 파이프라인
